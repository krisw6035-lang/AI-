# 产品需求文档（PRD）：房子魔法空间

> 版本：v0.1.0 | 日期：2026-07-03 | 作者：wu kris + Claude Code

## 1. 产品概述

### 1.1 产品名称

- 品牌：**新装**
- 产品：**房子魔法空间**
- 定位：AI 空间改造效果预览微信小程序

### 1.2 目标用户

| 用户类型 | 核心需求 | 使用场景 |
| --- | --- | --- |
| 装修业主 | 装修前快速预览不同风格的效果 | 翻新前想看看客厅改成新中式是什么样子 |
| 普通用户 | 好奇 AI 能做出什么效果 | 上传家里照片，看看不同风格的改造效果 |
| 建材/涂料门店 | 给客户快速出参考效果图 | 客户想看墙面换色效果，30 秒出图 |
| 室内设计师 | 快速验证设计方向 | 和客户沟通时快速生成风格参考 |
| 装修销售人员 | 帮助客户决策 | 给客户展示不同风格的改造可能性 |

### 1.3 核心价值主张

- **零门槛**：无需专业设计软件，只需上传照片
- **快速**：AI 自动生成，几十秒出效果图
- **多样**：9 种风格 × 5 种肌理 × 5 种色号 × 10 种软装
- **准确**：基于用户原图进行图生图改造，保留空间结构

## 2. 功能清单

### 2.1 核心用户路径

```
首页 → 选择场景（室内/室外）
  → 上传空间照片
  → 选择设计风格（9 选 1）
  → 选择艺术肌理（5 选 1）
  → 选择墙面色号（色卡单选）
  → [室内] 选择软装细节（多选）
  → 填写自定义要求（可选）
  → 点击生成（扣 10 积分）
  → 查看生成进度
  → 查看结果（原图/效果图对比）
```

### 2.2 功能模块

| 模块 | 功能点 | 优先级 |
| --- | --- | --- |
| 首页 | 品牌展示、双入口、操作流程、精选案例 | P0 |
| 图片上传 | 相册/拍照、预览、重传、格式校验 | P0 |
| 设计选择 | 风格/肌理/色号/软装/自定义要求 | P0 |
| AI 生成 | 图生图、任务状态机、异步轮询 | P0 |
| 积分系统 | 注册赠送、生成扣分、失败退回、流水 | P0 |
| 结果展示 | 原图/效果图对比、保存、重生成 | P0 |
| 微信登录 | code2Session 后端完成 | P0 |
| 历史记录 | 分页查询、仅查自己 | P1 |
| 案例展示 | 室内/室外案例、风格分类 | P1 |
| 探索发现 | 风格/色号/肌理推荐 | P2 |
| 关于/联系我们 | 品牌介绍、客服信息 | P2 |
| 用户协议 | 完整协议展示 | P0（审核必须） |
| 隐私政策 | 数据用途说明 | P0（审核必须） |
| 用户反馈 | 意见反馈提交 | P2 |

### 2.3 V1 不做

- 在线支付和积分购买
- 复杂后台管理系统
- 3D/VR/CAD/施工图
- 社交功能
- 多模型切换
- Web/H5/App 端

## 3. 验收标准

### 3.1 功能验收

1. 核心用户路径完整可用（首页→上传→设计→生成→结果）
2. 积分扣减与任务创建在同一事务中
3. 用户只能访问自己的数据
4. 所有敏感信息不在前端暴露

### 3.2 质量验收

1. 微信开发者工具可正常编译预览
2. 后端 Spring Boot 可本地启动
3. 核心接口返回统一结构
4. 生成任务轮询有超时机制

### 3.3 安全验收

1. API Key / AppSecret 不走前端
2. 鉴权在后端执行
3. 输入校验 + SQL 参数化
4. 用户协议 + 隐私政策完整
